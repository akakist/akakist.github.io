О Studio.h

Мы — команда разработчиков на C/C++, создаём высокопроизводительные бэкенды и распределённые системы на *nix с использованием фреймворка Eventron.

Eventron: Фреймворк для распределённых систем

Eventron — событийно-ориентированный фреймворк для финтеха, IoT, embedded. Основные особенности:

  • Производительность: 800k RPS (2 потока × ~400k, ab -n 1000000 -k -c 100) на Ubuntu 24.04 (6 CPU, 6 GB RAM). Сравнимо с Go (fasthttp) и Rust, в 2.5 раза быстрее Boost.
  • Событийная архитектура: Сервисы обрабатывают события внутри процесса или через кастомный RPC между процессами (300k RPS).
  • Изоляция: Сервисы — плагины (*.so), загружаемые on-demand, с изолированным кодом для распределённой разработки.
  • Многопоточность: Каждый сервис в отдельном потоке с собственным event loop, упрощающим разработку (однопоточное программирование).
  • Гибкость: Поддержка монолитов и распределённых систем. События с/без указания destination sockaddr.

mtjs: JS Runtime на Eventron

mtjs — компактный JS runtime (8 MB) на базе QuickJS, использующий Eventron. Достигает ~507k RPS (2 потока: 259k + 248k RPS) или 379k RPS в однопоточном режиме. Поддерживает Telnet-интерфейс и RPC (300k RPS). Идеален для финтеха, IoT, embedded.

Пример кода HTTP-сервера:

const http = mtjs.http;
http.createServer((req, res) => res.end("Hello, mtjs!")).listen(6012);

Запустите в Docker:

git clone https://github.com/akakist/mtjs.git
cd mtjs/mtjs
docker compose up

Примеры использования

Eventron и mtjs подходят для сложных распределённых систем:

  • Финтех: Высокопроизводительные бэкенды для транзакций (800k RPS).
  • IoT: Агрегация данных с датчиков, управление через GUI, кастомный RPC (300k RPS).
  • Embedded: Компактный бинарник mtjs (8 MB) для устройств с ограниченными ресурсами.
  • Кастомизация mtjs: Интеграция с Kafka, Redis, бизнес-логика в JS.

Пример: IoT-платформа с mtjs и Kafka для обработки 500k событий/сек.
