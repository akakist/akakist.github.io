About Studio.h

We are a team of C/C++ developers building high-performance backends and distributed systems on *nix using our Eventron framework.

Eventron: Framework for Distributed Systems

Eventron is an event-driven framework for fintech, IoT, and embedded. Key features:

  • Performance: 800k RPS (2 threads × ~400k, ab -n 1000000 -k -c 100) on Ubuntu 24.04 (6 CPU, 6 GB RAM). Comparable to Go (fasthttp) and Rust, 2.5x faster than Boost.
  • Event-driven architecture: Services process events within a process or via custom RPC across processes (300k RPS).
  • Isolation: Services are plugins (*.so), loaded on-demand, with isolated code for distributed development.
  • Multithreading: Each service runs in a single thread with its own event loop, enabling single-threaded programming.
  • Flexibility: Supports monoliths and distributed systems. Events with/without destination sockaddr.

mtjs: JS Runtime on Eventron

mtjs is a compact JS runtime (8 MB) based on QuickJS, powered by Eventron. Achieves ~507k RPS (2 threads: 259k + 248k RPS) or 379k RPS in single-thread mode. Supports Telnet interface and RPC (300k RPS). Ideal for fintech, IoT, embedded.

Example HTTP server code:

const http = mtjs.http;
http.createServer((req, res) => res.end("Hello, mtjs!")).listen(6012);

Run in Docker:

git clone https://github.com/akakist/mtjs.git
cd mtjs/mtjs
docker compose up

Use Cases

Eventron and mtjs are ideal for complex distributed systems:

  • Fintech: High-performance backends for transactions (800k RPS).
  • IoT: Data aggregation from sensors, GUI control, custom RPC (300k RPS).
  • Embedded: Compact mtjs binary (8 MB) for resource-constrained devices.
  • mtjs Customization: Integration with Kafka, Redis, business logic in JS.

Example: Deployed IoT platform with mtjs and Kafka for 500k events/sec.
